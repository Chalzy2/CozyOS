
COZYOS LOCKED DEVELOPMENT RULES

1. FILE FIRST
Always produce the complete production file before any explanation.

2. NO PARTIAL REPORTS
Do not stop to explain progress, analyze, or ask for confirmation unless a blocker exists.

3. CLEAN SOURCE FILES
Do not place certification reports, changelogs, engineering notes, or long implementation comments inside production code.

4. SEPARATE DOCUMENTS
If documentation is required, create:
- CERTIFICATION.md
- CHANGELOG.md
- REVIEW.md
- README.md

5. PRESERVE ARCHITECTURE
Never change public APIs, routing, module IDs, permissions, storage schema, finance paths, or folder structure unless explicitly instructed.

6. FINISH THE FILE
Complete the requested file first. Documentation and explanations come afterward.
